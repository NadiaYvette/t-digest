# t-digest — 스트리밍 분위수 추정 데이터 구조

## 개요

**t-digest**는 데이터 스트림에서 분위수(백분위수)를 효율적으로 추정하기 위한 확률적 데이터 구조입니다. Ted Dunning이 고안한 이 알고리즘은 대량의 데이터를 메모리에 모두 저장하지 않고도 스트리밍 방식으로 데이터를 순차적으로 처리하면서 정확한 분위수 추정을 제공합니다.

t-digest의 가장 뛰어난 특성은 분포의 꼬리(tail) 부분에서 특히 높은 정확도를 보인다는 것입니다. 중앙값 근처의 정확도는 다소 낮을 수 있지만, 99번째 백분위수나 99.9번째 백분위수와 같은 극단적인 분위수를 추정할 때 매우 높은 정확도를 발휘합니다. 이는 SLA 모니터링이나 지연 시간 분석과 같은 실무에서 매우 중요한 성질입니다.

내부적으로 t-digest는 데이터 포인트를 "센트로이드(centroid)"라 불리는 클러스터로 집계하여 저장합니다. 메모리 사용량은 압축 매개변수 δ에 의해 제어되며, 입력 데이터의 총 개수에 의존하지 않는 O(δ)의 유한 메모리로 동작합니다.

## 주요 특성

- **스트리밍/온라인 처리** — 데이터를 하나씩 순차적으로 추가할 수 있으며, 모든 데이터를 메모리에 보관할 필요가 없습니다
- **유한 메모리 O(δ)** — 메모리 사용량은 압축 매개변수 δ에만 의존하며, 데이터 양과 무관하게 일정합니다
- **꼬리 정확도** — 분포의 양 끝(0% 근처 및 100% 근처)에서 특히 높은 정확도를 달성합니다
- **병합 가능** — 여러 t-digest를 결합할 수 있어 분산 시스템에서의 활용에 적합합니다

## 스케일 함수 개념

t-digest의 핵심에는 "스케일 함수"가 있습니다. 스케일 함수는 분포의 각 위치에서 센트로이드가 얼마나 커질 수 있는지를 제어합니다. 분포의 중앙 부근에서는 센트로이드가 크게(많은 데이터 포인트를 포함하여) 성장하는 것을 허용하고, 꼬리 부분에서는 작은 센트로이드만 허용합니다. 이 메커니즘을 통해 꼬리 부분에 더 많은 센트로이드가 배치되어 꼬리에서의 추정 정확도가 향상됩니다.

## 사용 예시 (의사 코드)

```
# t-digest 생성 (압축 매개변수 δ = 100)
td = TDigest.new(delta: 100)

# 데이터 포인트를 순차적으로 추가
td.add(1.0)
td.add(2.5)
td.add(3.7)
td.add(100.0)
td.add(0.01)

# 대량의 데이터도 문제없이 처리 가능
for value in data_stream:
    td.add(value)

# 분위수 조회
median    = td.quantile(0.5)    # 중앙값
p99       = td.quantile(0.99)   # 99번째 백분위수
p999      = td.quantile(0.999)  # 99.9번째 백분위수

# 역방향 조회: 특정 값의 CDF 값을 얻기
cdf_value = td.cdf(42.0)        # 42.0 이하인 데이터의 비율
```
