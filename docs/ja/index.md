# t-digest — ストリーミング分位数推定データ構造

## 概要

**t-digest** は、データストリームから分位数（パーセンタイル）を効率的に推定するための確率的データ構造です。Ted Dunning によって考案されたこのアルゴリズムは、大量のデータを一度にメモリに保持することなく、ストリーミング方式でデータを逐次処理しながら正確な分位数推定を実現します。

t-digest の最も優れた特性は、分布の裾（テール）における精度が特に高いことです。中央値付近の精度は若干劣りますが、99パーセンタイルや99.9パーセンタイルといった極端な分位数の推定において非常に高い精度を発揮します。これは、SLA監視やレイテンシ分析といった実用的な場面で非常に重要な性質です。

内部的には、t-digest はデータポイントを「セントロイド」と呼ばれるクラスタに集約して保持します。使用するメモリ量は圧縮パラメータ δ によって制御され、入力データの総数に依存しない O(δ) の有界メモリで動作します。

## 主な特性

- **ストリーミング／オンライン処理** — データを一つずつ逐次的に追加でき、すべてのデータをメモリに保持する必要がありません
- **有界メモリ O(δ)** — メモリ使用量は圧縮パラメータ δ のみに依存し、データ量に関わらず一定です
- **裾の精度** — 分布の両端（0%付近および100%付近）で特に高い精度を実現します
- **マージ可能** — 複数の t-digest を結合でき、分散システムでの利用に適しています

## スケール関数の概念

t-digest の核心にあるのが「スケール関数」です。スケール関数は、分布の各位置においてセントロイドがどの程度の大きさまで成長できるかを制御します。分布の中央付近ではセントロイドが大きく（多くのデータポイントを含み）なることを許容し、裾の部分では小さなセントロイドのみを許可します。この仕組みにより、裾の部分でより多くのセントロイドが配置され、結果として裾における推定精度が向上します。

## 使用例（擬似コード）

```
# t-digestを作成（圧縮パラメータ δ = 100）
td = TDigest.new(delta: 100)

# データポイントを逐次追加
td.add(1.0)
td.add(2.5)
td.add(3.7)
td.add(100.0)
td.add(0.01)

# 大量のデータも問題なく処理可能
for value in data_stream:
    td.add(value)

# 分位数を照会
median    = td.quantile(0.5)    # 中央値
p99       = td.quantile(0.99)   # 99パーセンタイル
p999      = td.quantile(0.999)  # 99.9パーセンタイル

# 逆クエリ：あるデータ値のCDF値を取得
cdf_value = td.cdf(42.0)        # 42.0以下のデータの割合
```
