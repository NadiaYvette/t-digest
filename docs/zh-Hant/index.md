# t-digest — 串流分位數估計資料結構

## 概述

**t-digest** 是一種用於從資料串流中高效估計分位數（百分位數）的機率性資料結構。此演算法由 Ted Dunning 提出，能夠在不將全部資料保存於記憶體中的情況下，以串流方式逐步處理資料並提供準確的分位數估計。

t-digest 最突出的特性是在分佈的尾部（tail）具有特別高的精確度。雖然在中位數附近的精確度略低，但在估計第 99 百分位數或第 99.9 百分位數等極端分位數時，表現出極高的準確性。這一特性在 SLA 監控和延遲分析等實際場景中非常重要。

在內部實作上，t-digest 將資料點聚合為稱作「質心（centroid）」的叢集進行儲存。記憶體使用量由壓縮參數 δ 控制，以 O(δ) 的有界記憶體運行，不依賴輸入資料的總量。

## 核心特性

- **串流／線上處理** — 可以逐個添加資料點，無需將所有資料保存在記憶體中
- **有界記憶體 O(δ)** — 記憶體使用量僅取決於壓縮參數 δ，與資料量無關
- **尾部精確度** — 在分佈的兩端（接近 0% 和 100%）實現特別高的精確度
- **可合併** — 多個 t-digest 可以合併，適用於分散式系統

## 縮放函數概念

t-digest 的核心是「縮放函數（scale function）」。縮放函數控制在分佈的每個位置上質心可以增長到多大。在分佈的中央部分，允許質心較大（包含更多資料點）；在尾部，只允許較小的質心。透過這一機制，尾部區域會配置更多的質心，從而提升尾部的估計精確度。

## 使用範例（虛擬碼）

```
# 建立 t-digest（壓縮參數 δ = 100）
td = TDigest.new(delta: 100)

# 逐步添加資料點
td.add(1.0)
td.add(2.5)
td.add(3.7)
td.add(100.0)
td.add(0.01)

# 可以輕鬆處理大量資料
for value in data_stream:
    td.add(value)

# 查詢分位數
median    = td.quantile(0.5)    # 中位數
p99       = td.quantile(0.99)   # 第99百分位數
p999      = td.quantile(0.999)  # 第99.9百分位數

# 反向查詢：取得某個值的 CDF 值
cdf_value = td.cdf(42.0)        # 小於等於42.0的資料佔比
```
