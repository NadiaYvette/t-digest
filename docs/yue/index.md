# t-digest — 串流分位數估算資料結構

## 概覽

**t-digest** 係一種用嚟從資料串流度高效噉估算分位數（百分位數）嘅機率性資料結構。呢個演算法係 Ted Dunning 發明嘅，可以用串流方式逐個噉處理數據，唔使將所有數據儲喺記憶體入面，同時提供準確嘅分位數估算。

t-digest 最出色嘅特性係喺分佈嘅尾部（tail）有特別高嘅精確度。雖然喺中位數附近嘅精確度可能稍為低啲，但係喺估算極端分位數（好似第 99 百分位數或者第 99.9 百分位數）嗰陣，精確度非常之高。呢個特性喺 SLA 監控同延遲分析等實際應用入面非常重要。

喺內部實作上，t-digest 將數據點聚合成叫做「質心（centroid）」嘅叢集嚟儲存。記憶體使用量由壓縮參數 δ 控制，以 O(δ) 嘅有界記憶體運行，唔受輸入資料總量影響。

## 主要特性

- **串流／線上處理** — 可以逐個噉加入數據點，唔使將所有資料儲喺記憶體入面
- **有界記憶體 O(δ)** — 記憶體使用量淨係取決於壓縮參數 δ，同資料量無關
- **尾部精確度** — 喺分佈嘅兩端（接近 0% 同 100%）達到特別高嘅精確度
- **可以合併** — 多個 t-digest 可以合併，適合分散式系統使用

## 縮放函數概念

t-digest 嘅核心係「縮放函數（scale function）」。縮放函數控制喺分佈嘅每個位置上質心可以增長到幾大。喺分佈嘅中央部分，容許質心比較大（包含更多數據點）；喺尾部，淨係容許細嘅質心。透過呢個機制，尾部區域會配置更多嘅質心，從而提升尾部嘅估算精確度。

## 使用範例（虛擬碼）

```
# 建立 t-digest（壓縮參數 δ = 100）
td = TDigest.new(delta: 100)

# 逐個加入數據點
td.add(1.0)
td.add(2.5)
td.add(3.7)
td.add(100.0)
td.add(0.01)

# 處理大量數據都冇問題
for value in data_stream:
    td.add(value)

# 查詢分位數
median    = td.quantile(0.5)    # 中位數
p99       = td.quantile(0.99)   # 第99百分位數
p999      = td.quantile(0.999)  # 第99.9百分位數

# 反向查詢：攞某個值嘅 CDF 值
cdf_value = td.cdf(42.0)        # 細過或者等於42.0嘅資料佔比
```
