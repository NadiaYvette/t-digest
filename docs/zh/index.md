# t-digest — 流式分位数估计数据结构

## 概述

**t-digest** 是一种用于从数据流中高效估计分位数（百分位数）的概率性数据结构。该算法由 Ted Dunning 提出，能够在不将全部数据保存在内存中的情况下，以流式方式逐步处理数据并提供准确的分位数估计。

t-digest 最突出的特性是在分布的尾部（tail）具有特别高的精度。虽然在中位数附近的精度略低，但在估计第 99 百分位数或第 99.9 百分位数等极端分位数时，它表现出极高的准确性。这一特性在 SLA 监控和延迟分析等实际场景中非常重要。

在内部实现上，t-digest 将数据点聚合为称作"质心（centroid）"的簇进行存储。内存使用量由压缩参数 δ 控制，以 O(δ) 的有界内存运行，不依赖于输入数据的总量。

## 核心特性

- **流式／在线处理** — 可以逐个添加数据点，无需将所有数据保存在内存中
- **有界内存 O(δ)** — 内存使用量仅取决于压缩参数 δ，与数据量无关
- **尾部精度** — 在分布的两端（接近 0% 和 100%）实现特别高的精度
- **可合并** — 多个 t-digest 可以合并，适用于分布式系统

## 缩放函数概念

t-digest 的核心是"缩放函数（scale function）"。缩放函数控制在分布的每个位置上质心可以增长到多大。在分布的中央部分，允许质心较大（包含更多数据点）；在尾部，只允许较小的质心。通过这一机制，尾部区域会配置更多的质心，从而提高尾部的估计精度。

## 使用示例（伪代码）

```
# 创建 t-digest（压缩参数 δ = 100）
td = TDigest.new(delta: 100)

# 逐步添加数据点
td.add(1.0)
td.add(2.5)
td.add(3.7)
td.add(100.0)
td.add(0.01)

# 可以无压力地处理大量数据
for value in data_stream:
    td.add(value)

# 查询分位数
median    = td.quantile(0.5)    # 中位数
p99       = td.quantile(0.99)   # 第99百分位数
p999      = td.quantile(0.999)  # 第99.9百分位数

# 反向查询：获取某个值的 CDF 值
cdf_value = td.cdf(42.0)        # 小于等于42.0的数据占比
```
