# t-digest — நிகழ்நேரத் தரவுக்கான நூற்றுமானக் கணிப்பு

## கண்ணோட்டம்

**t-digest** என்பது தரவுப் பாய்வில் இருந்து நூற்றுமானங்களை (quantiles) திறம்படக் கணிக்கும் ஒரு நிகழ்தகவு புள்ளியியல் கட்டமைப்பு ஆகும். இதை Ted Dunning உருவாக்கினார். கோடிக்கணக்கான மதிப்புகள் வரும்போது இடைநிலை, 99-ஆவது சதவிகிதம் அல்லது வேறு எந்த நூற்றுமானத்தையும் அறிய அனைத்து மதிப்புகளையும் நினைவகத்தில் வைப்பது நடைமுறைச் சாத்தியமில்லை. t-digest இந்தச் சிக்கலைத் தீர்க்கிறது — இது வரையறுக்கப்பட்ட நினைவகத்தில் இயங்குகிறது, அதே சமயம் மிகத் துல்லியமான முடிவுகளை அளிக்கிறது.

t-digest-ன் சிறப்பியல்பு என்னவென்றால், இது பரவலின் வால்களில் (tails) மிகவும் துல்லியமானது. அதாவது 99-ஆவது அல்லது 99.9-ஆவது சதவிகிதத்தின் கணிப்பு நடுப்பகுதியை விட நம்பகமானது. இந்தப் பண்பு தாமதக் கண்காணிப்பு, வழக்கத்திற்கு மாறான நிகழ்வுகளைக் கண்டறிதல் மற்றும் சேவை-நிலை ஒப்பந்த (SLA) அளவீடு ஆகியவற்றுக்கு மிகவும் பயனுள்ளது.

இந்தத் திட்டத்தில் t-digest எட்டு நிரலாக்க மொழிகளில் கிடைக்கிறது: Ruby, Haskell, Common Lisp, Scheme, Standard ML, Ada, Prolog மற்றும் Mercury.

## முக்கிய பண்புகள்

- **நிகழ்நேர/இணையநிலை கணக்கீடு** — மதிப்புகளை ஒவ்வொன்றாக அல்லது தொகுதிகளாகச் சேர்க்கலாம்; எல்லாத் தரவும் ஒரே நேரத்தில் இருக்க வேண்டியதில்லை
- **வரையறுக்கப்பட்ட நினைவகம் O(δ)** — நினைவகப் பயன்பாடு தரவின் அளவைச் சார்ந்தது அல்ல, அமுக்க அளவுரு δ (டெல்டா) சார்ந்தது; பொதுவாக δ=100 போதுமானது
- **வால்-துல்லியம்** — பரவலின் கடைசி முனைகளில் (0.01-ஆவது அல்லது 99.99-ஆவது சதவிகிதம்) துல்லியம் சிறந்ததாக இருக்கும், இது நடைமுறை பயன்பாடுகளில் மிக முக்கியம்
- **இணைக்கத்தக்கது** — இரண்டு t-digest கட்டமைப்புகளை ஒன்றாக இணைக்கலாம், இது விநியோகிக்கப்பட்ட அமைப்புகளுக்கு ஏற்றது

## அளவுச்சார்பின் கருத்துரு

t-digest எங்கு அதிகத் துல்லியம் வைக்க வேண்டும், எங்கு குறைவாக வைக்கலாம் என்பதை முடிவு செய்ய **அளவுச்சார்பு** (scale function) பயன்படுத்துகிறது. ஒரு அளவுகோலை நினைத்துப் பாருங்கள் — அதன் இரு முனைகளிலும் கோடுகள் மிக நெருக்கமாகவும், நடுவில் தூரமாகவும் உள்ளன. அளவுச்சார்பு இதையே செய்கிறது — பரவலின் முனைகளில் சிறு சிறு குழுக்களை (centroids) உருவாக்கி அங்கு துல்லியத்தை அதிகரிக்கிறது, நடுவில் பெரிய குழுக்களை உருவாக்குகிறது — அங்கு சிறிய பிழை ஏற்றுக்கொள்ளத்தக்கது.

## எடுத்துக்காட்டு: போலிக்குறிமுறை (pseudocode)

```
# t-digest உருவாக்குக (அமுக்க அளவுரு δ = 100)
td = TDigest.new(delta: 100)

# மதிப்புகளைச் சேர்க்க
for value in data_stream:
    td.add(value)

# நூற்றுமானங்களை வினவுக
median    = td.quantile(0.5)    # இடைநிலை
p95       = td.quantile(0.95)   # 95-ஆவது சதவிகிதம்
p99       = td.quantile(0.99)   # 99-ஆவது சதவிகிதம்
p999      = td.quantile(0.999)  # 99.9-ஆவது சதவிகிதம்

# ஒரு மதிப்பின் சதவிகித வரிசையை அறிக
rank = td.cdf(150.0)  # 150-க்குக் குறைவான மதிப்புகளின் விகிதம்
```
